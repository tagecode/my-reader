## 2. 产品概述

### 2.1 产品定位

本产品是一款跨平台、纯本地优先的桌面电子阅读器，面向拥有大量本地电子书、PDF、漫画和资料文档的用户。产品默认不依赖账号、不强制联网、不上传书籍内容，通过现代化界面、稳定阅读体验和广泛格式兼容，提供一个可长期使用的本地个人阅读中心。

### 2.2 产品目标

1. 建立一个稳定可用的本地书库，支持导入、管理、搜索和打开本地书籍。
2. 优先支持 EPUB、PDF、TXT 三类核心格式，覆盖最常见阅读需求。
3. 提供舒适的阅读体验，包括进度记忆、目录跳转、主题切换、基础排版调整。
4. 在后续阶段扩展 MOBI、AZW3、FB2、CBZ、DJVU、CHM 等格式。
5. 通过全文搜索、笔记高亮、OCR、TTS、WebDAV 同步等功能形成差异化。
6. 坚持本地优先和隐私优先，所有在线能力均为可选增强。

### 2.3 成功标准

- 用户可以在无网络环境下完成导入、管理和阅读 EPUB、PDF、TXT。
- 用户重新打开应用或书籍后，阅读进度可以准确恢复。
- 普通大小书籍打开和阅读过程不卡死、不明显阻塞主界面。
- 用户能够通过现代化书库界面快速找到最近阅读和已导入书籍。
- 用户可以根据阅读习惯调整主题、字号和基础阅读布局。
- 核心阅读数据默认只保存在本机。

## 3. 背景与问题

### 3.1 用户痛点

- 本地电子书格式分散，用户需要多个工具分别阅读 EPUB、PDF、TXT、漫画和旧格式电子书。
- 传统桌面阅读器界面相对陈旧，交互体验不符合现代应用习惯。
- 云端阅读产品通常要求登录、上传文件或依赖网络，不适合重视隐私和离线阅读的用户。
- 大型 PDF、扫描版文档、长篇 TXT 容易出现卡顿、搜索困难、目录缺失等问题。
- 用户的阅读进度、笔记和书签通常分散在不同应用内，难以统一管理。

### 3.2 产品机会

Electron + React + TypeScript + shadcn/ui 的技术组合可以较快构建跨平台桌面应用，并通过 `foliate-js`、`pdf.js`、SQLite FTS5 等成熟方案覆盖核心阅读能力。产品可以在“本地优先 + 现代 UI + 多格式支持”上形成明确定位，区别于传统工具型阅读器和云端内容平台。

## 4. 目标用户与使用场景

### 4.1 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---|---|---|
| 本地书库用户 | 拥有大量 EPUB、PDF、TXT、MOBI 等文件 | 统一管理和阅读多格式书籍 |
| 学习研究用户 | 阅读技术书、论文、资料文档 | PDF 性能、搜索、笔记、高亮 |
| 小说阅读用户 | 阅读长篇 TXT、EPUB、网络小说整理文件 | 编码兼容、章节识别、舒适排版 |
| 漫画用户 | 阅读 CBZ、CBR、图片合集 | 图片排序、双页、阅读方向 |
| 隐私敏感用户 | 不希望上传书籍和阅读记录 | 完全本地、可控同步 |

### 4.2 核心用户场景

#### 场景 1：首次导入本地书籍

用户安装应用后，将一批 EPUB、PDF、TXT 拖入窗口。应用自动识别格式、提取标题、作者、封面和文件信息，并将书籍展示在书库中。导入失败的文件需要明确说明原因。

#### 场景 2：继续上次阅读

用户打开应用后，在书库首页看到最近阅读的书籍，点击后直接恢复到上次阅读位置，并使用上次保存的主题和排版设置。

#### 场景 3：阅读 PDF 技术书

用户打开一本较大的 PDF 技术书，应用按需渲染当前页面，支持缩放、页码跳转、目录跳转和进度显示。阅读过程中应用不应因为一次性渲染过多页面而卡死。

#### 场景 4：阅读长篇 TXT 小说

用户导入 GBK 或 GB18030 编码的 TXT 小说，应用自动识别编码并正常显示中文内容，尽可能识别章节目录。用户可以调整字号、行距、边距和主题。

#### 场景 5：本地搜索资料

用户希望在书库中查找某本书，或在某本书内搜索关键词。应用应支持按标题、作者搜索书库，并在核心体验阶段支持书内搜索和全文索引。

## 5. 产品范围

### 5.1 MVP 范围

MVP 目标是完成一个真正可用的本地阅读器，重点支持导入、书库、EPUB/TXT/PDF 阅读、进度记忆和基础设置。

| 模块 | MVP 范围 |
|---|---|
| 书库 | 文件导入、拖拽导入、基础元数据、网格/列表展示 |
| 格式 | EPUB、TXT、PDF |
| 阅读 | 打开书籍、翻页/滚动、目录跳转、进度显示、进度恢复 |
| 设置 | 日夜主题、基础阅读设置 |
| 存储 | SQLite 保存书库、进度、设置 |
| 跨平台 | macOS、Windows、Linux 基础打包 |
| 隐私 | 所有核心数据默认本地保存 |

### 5.2 非 MVP 范围

以下功能不进入首版 MVP，但可作为后续版本规划：

- 笔记、高亮、OCR、TTS、翻译、AI 摘要。
- WebDAV 同步、多设备同步、端到端加密同步。
- OPDS、在线书城、内容分发。
- 插件系统、主题市场。
- 移动端。
- DRM 破解或绕过。
- 复杂排版编辑器。

## 8. UI 与交互要求

### 8.1 信息架构

产品主界面由以下区域组成：

- 书库首页：展示所有导入书籍、最近阅读、筛选和搜索入口。
- 阅读界面：承载电子书正文、工具栏、目录、进度、设置入口。
- 设置中心：管理主题、阅读偏好、书库路径、快捷键、缓存、同步等配置。
- 侧边栏/抽屉：展示目录、书签、笔记、搜索结果。

### 8.2 关键界面

| 页面 | 主要内容 | 关键要求 |
|---|---|---|
| 书库页 | 书籍列表、筛选、搜索、导入入口 | 清晰、快速、适合大量书籍 |
| 阅读页 | 正文、进度、目录、工具栏 | 低干扰、可专注、可自定义 |
| 设置页 | 全局偏好与高级配置 | 分类明确、危险操作有确认 |
| 导入反馈 | 成功、失败、重复、解析中 | 状态明确、可追踪 |

### 8.3 组件要求

- 使用 shadcn/ui `Card` 构建书籍卡片。
- 使用 shadcn/ui `Dialog` 构建设置与导入弹窗。
- 使用 shadcn/ui `Sidebar` 构建书库侧边栏。
- 使用 shadcn/ui `Sheet` 构建目录、书签、笔记侧栏。
- 使用 shadcn/ui `DropdownMenu` 和 `ContextMenu` 构建操作菜单。
- 使用 shadcn/ui `Command` 构建命令面板。
- 使用 shadcn/ui `Slider` 构建字号、行距、亮度调节。
- 使用 shadcn/ui `Switch` 构建开关项。
- 使用 shadcn/ui `Toaster` 提供操作反馈。

### 8.4 交互原则

- 阅读界面默认低干扰，工具栏可自动隐藏或手动收起。
- 所有耗时任务必须显示进度或状态。
- 所有失败状态必须给出可理解原因。
- 涉及删除、清空、覆盖、同步冲突等危险操作必须二次确认。
- 在线能力默认关闭，启用前需要明确提示隐私影响。

## 9. 非功能需求

### 9.1 性能

| 编号 | 需求 |
|---|---|
| NFR-PERF-001 | 应用冷启动不应出现长时间白屏或无响应。 |
| NFR-PERF-002 | 导入普通 EPUB、TXT、PDF 时界面不可冻结。 |
| NFR-PERF-003 | 大 PDF 必须按需渲染，不允许一次性渲染全部页面。 |
| NFR-PERF-004 | 大 TXT 必须支持分块读取或分页索引。 |
| NFR-PERF-005 | 大书库列表应使用虚拟滚动或等效优化。 |
| NFR-PERF-006 | 全文索引必须在后台执行，不阻塞阅读。 |
| NFR-PERF-007 | Electron 安装包体积目标控制在 80-120MB，允许因 OCR 等高级能力增加可选组件。 |

### 9.2 稳定性

| 编号 | 需求 |
|---|---|
| NFR-STAB-001 | 单本书解析失败不应导致应用崩溃。 |
| NFR-STAB-002 | 文件丢失、权限不足、格式不支持时需要可恢复。 |
| NFR-STAB-003 | 后台任务失败需要记录错误并反馈给用户。 |
| NFR-STAB-004 | 应用异常退出后不应损坏书库数据库。 |

### 9.3 安全与隐私

| 编号 | 需求 |
|---|---|
| NFR-SEC-001 | Electron 渲染进程禁用不必要的 Node.js 能力。 |
| NFR-SEC-002 | 使用 `preload` 和 `contextBridge` 暴露最小 IPC API。 |
| NFR-SEC-003 | 本地文件访问必须通过主进程受控接口完成。 |
| NFR-SEC-004 | EPUB HTML 渲染需要沙箱隔离，禁止书籍内容访问应用能力。 |
| NFR-SEC-005 | 所有在线能力默认关闭。 |
| NFR-SEC-006 | 首次启用在线翻译、在线 TTS、在线 AI 前必须提示隐私影响。 |
| NFR-SEC-007 | 用户可以导出和删除本地用户数据。 |

### 9.4 可访问性

| 编号 | 需求 |
|---|---|
| NFR-A11Y-001 | 基础界面支持键盘导航。 |
| NFR-A11Y-002 | 重要按钮和菜单提供可访问名称。 |
| NFR-A11Y-003 | 阅读器支持高对比度主题。 |
| NFR-A11Y-004 | 字号、行距、边距可调。 |

### 9.5 可维护性

| 编号 | 需求 |
|---|---|
| NFR-MAINT-001 | 格式解析、阅读渲染、书库管理、搜索索引、同步能力需要模块化。 |
| NFR-MAINT-002 | 主进程与渲染进程通过明确 IPC 契约通信。 |
| NFR-MAINT-003 | 核心数据模型需要保持迁移能力。 |
| NFR-MAINT-004 | 解析器失败原因应可记录、可定位。 |

### 10.2 数据保存原则

- 数据结构应预留版本号，支持后续迁移。

## 11. 里程碑规划

### 11.1 Phase 1：MVP 基础版

目标：完成可用的本地阅读器。

- SQLite 本地数据库。
- 文件导入和拖拽导入。

- 阅读进度记录。

- 多主题。

- DJVU、CBR、CHM 支持。

- 数据备份与恢复。

### 11.4 Phase 4：长期生态版

目标：提升扩展性和长期价值。

- OPDS。
- 国际化完善。
- 更多平台优化。
- 移动端可行性探索。

## 13. 风险与约束

| 风险 | 影响 | 应对策略 |
|---|---|---|
| PDF 大文件卡顿 | 影响核心阅读体验 | 从 MVP 阶段引入 Worker、按需渲染和缓存策略 |
| EPUB 复杂排版兼容性 | 影响格式可靠性 | 优先使用 `foliate-js`，避免从零实现完整排版引擎 |
| MOBI/AZW3 兼容性不稳定 | 影响格式覆盖承诺 | 使用 `foliate-js` 优先解析，后续引入 Calibre CLI 兜底 |
| OCR 实现成本高 | 影响高级功能周期 | 放入 P2 阶段，先做单页 OCR，再扩展整书 OCR |
| Electron 安装包体积大 | 影响下载安装体验 | 通过按需加载、压缩、移除无关资源优化，接受合理体积 |
| 在线能力带来隐私风险 | 影响产品定位 | 默认关闭，启用前明确提示，优先本地方案 |
| 跨平台差异 | 影响稳定性 | 尽早建立 macOS、Windows、Linux 打包和测试流程 |

## 14. 不做事项

- 不做在线书城和内容分发。
- 不做用户账号系统作为核心依赖。
- 不做社交、评论、社区、排行榜。
- 不做云端强制同步。
- 不做 DRM 破解或版权保护绕过。
- 不做复杂排版编辑器。
- 不以移动端作为首发平台。

- 应用名称、品牌视觉和图标风格需要单独确定。
