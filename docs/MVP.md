# 摸鱼阅读器（My Reader）MVP 功能清单与开发任务

## 1. MVP 目标

MVP 目标是交付一个真正可用的纯本地桌面电子阅读器，让用户可以在无网络环境下完成本地书籍导入、书库管理、EPUB/TXT/PDF 阅读、阅读进度恢复和基础阅读设置。

首版不追求完整格式覆盖和高级阅读增强，而是优先验证以下核心闭环：

1. 用户可以导入本地电子书。
2. 用户可以在书库中看到导入的书籍。
3. 用户可以打开 EPUB、TXT、PDF 并完成基础阅读。
4. 用户关闭应用后再次打开，阅读进度可以恢复。
5. 用户可以切换日间/夜间主题，并进行基础阅读设置。
6. 所有核心数据默认保存在本机。

## 2. MVP 范围

| 模块 | MVP 范围 | 对应 PRD |
|---|---|---|
| 项目基础 | Electron + React + TypeScript + Vite + shadcn/ui + Tailwind CSS | Phase 1 |
| 本地存储 | SQLite 保存书库、阅读进度和设置 | DATA-001、DATA-002 |
| 书籍导入 | 文件选择导入、拖拽导入、格式识别、基础元数据提取 | LIB-001 至 LIB-004 |
| 书库展示 | 网格视图、列表视图、书籍卡片、基础搜索、打开/移除书籍 | LIB-011 至 LIB-013、LIB-020、LIB-021、SRCH-001 |
| EPUB 阅读 | 元数据、目录、正文渲染、目录跳转 | FMT-001、EPUB-001 至 EPUB-003、RD-005 |
| TXT 阅读 | 编码识别、手动编码选择、大文件分块读取 | FMT-002、TXT-001 至 TXT-003 |
| PDF 阅读 | PDF 加载、渲染、滚动、缩放、页码跳转、Worker 配置 | FMT-003、PDF-001 至 PDF-005 |
| 阅读体验 | 打开书籍、翻页/滚动、进度显示、进度恢复、返回书库 | RD-001 至 RD-007、RD-024、RD-025 |
| 主题设置 | 日间模式、夜间模式、基础阅读默认设置 | RD-013、SET-001、SET-002 |
| 跨平台 | macOS、Windows、Linux 基础打包，文件打开方式关联 | SYS-001、SYS-002 |
| 安全隐私 | 最小 IPC、本地文件受控访问、核心数据本地保存 | NFR-SEC-001 至 NFR-SEC-005 |

## 3. MVP 不包含的功能

以下功能不进入 MVP，避免首版范围失控：

- MOBI、AZW3、FB2、CBZ、CBR、DJVU、CHM 等扩展格式。
- 书内搜索、全文索引、SQLite FTS5。
- 书签、笔记、高亮。
- OCR、TTS、翻译、词典、AI 摘要。
- WebDAV 同步、多设备同步、端到端加密。
- Calibre CLI 兜底转换。
- 多窗口、分栏阅读、专注模式、双页对开。
- 插件系统、主题市场、移动端。
- DRM 破解或版权保护绕过。

## 7. MVP 技术约束

- Electron 渲染进程禁用不必要的 Node.js 能力。
- 主进程负责本地文件读取和数据库访问。
- 渲染进程通过 `preload` 暴露的最小 API 访问本地能力。
- EPUB、TXT、PDF 解析失败不能导致应用崩溃。
- 导入、解析和渲染过程不能长时间阻塞 UI。
- PDF 必须配置 Worker，避免主线程承担重渲染任务。
- TXT 必须考虑中文编码和大文件读取。
- 书籍源文件默认不复制，优先保存原始文件路径。
- 阅读进度和设置必须持久化到本地。
- MVP 不接入在线服务。

## 8. MVP 风险与应对

| 风险 | 影响 | MVP 应对 |
|---|---|---|
| EPUB 渲染兼容性不足 | EPUB 阅读体验不稳定 | 优先使用成熟库，首版只保证基础阅读、目录和进度 |
| PDF 大文件卡顿 | 影响核心体验 | 配置 Worker，避免一次性渲染全部页面 |
| TXT 中文编码识别失败 | 中文小说无法阅读 | 自动识别失败时允许手动选择编码 |
| SQLite 原生依赖打包复杂 | 影响跨平台打包 | 尽早在 Sprint 1 验证数据库方案和打包兼容性 |
| Electron 安全配置疏漏 | 带来本地文件安全风险 | 从项目初始化阶段建立最小 IPC 和受控文件访问 |
| MVP 范围膨胀 | 延迟交付 | 严格排除 P1/P2/P3 高级功能 |

## 9. 建议开发顺序

1. 先完成项目骨架、IPC 和数据库，不先做阅读器 UI。
2. 再完成书籍导入和书库展示，形成文件进入系统的入口。
3. 然后建立统一阅读页和进度保存机制。
4. 按 EPUB、TXT、PDF 的顺序实现具体格式阅读。
5. 最后补齐主题、设置、打包、安全检查和验收。

这个顺序可以尽早验证最关键的数据流：本地文件 -> 书库记录 -> 阅读器 -> 阅读进度 -> 本地恢复。

## 10. 后续版本衔接

MVP 完成后，优先进入 Phase 2 核心体验版，建议顺序为：

1. 书库排序、筛选、最近阅读。
2. 书签和目录侧栏增强。
3. 书内搜索和全文索引。
4. 字体、行距、边距、多主题。
5. MOBI、AZW3、FB2、CBZ 支持。
6. PDF 性能进一步优化。
